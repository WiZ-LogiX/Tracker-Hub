## ❓ Missing Information (material gaps)

- **No SLA/RTO/RPO** — no documentation for backups, disaster recovery, or recovery time.
- **No documented effective RBAC** — roles (owner/admin/sales/worker/viewer) are defined, but each role's permissions at the UI/action level are not documented.
- **No complete data flow for quote→order→invoice** — steps are described but transaction boundaries are not.
- **No n8n webhook contract** — payload schema, auth (HMAC?), and retry policy.
- **No cost model** — R2 + Workers + DB cost under growth.
- **No environments matrix** — dev/staging/prod and the differences between them.

## 🌫️ Ambiguous / Weak-Evidence Areas

- **Neon readiness:** "checkNeonConnection works" does not mean the migration happened — weak evidence.
- **"production tracking 100%":** functionally complete but no evidence of load/realtime testing.
- **i18n "supports ar/en/fr":** reality is ar/fr partial — description is broader than implementation.

## 🧩 Undocumented Assumptions

- **Assumption that RLS alone is sufficient for isolation** — a dangerous, unstated assumption.
- **Assumption that the catalog is global (not tenant-scoped)** — hence it uses supabaseAdmin, but neither the rationale nor the risk is documented.
- **Assumption that window.print is sufficient for invoices** — inappropriate for a commercial SaaS.
- **Assumption that synthetic emails (@pelecanon.local) are a permanent solution** — a temporary hack the auth is built on.
